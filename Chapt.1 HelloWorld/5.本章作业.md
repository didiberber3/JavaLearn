   1. ```
      JDK、JRE 和 JVM 是 Java 相关的三个核心概念，它们之间有明确的层次关系：
      
      1. **JDK（Java Development Kit）**：
         - **作用**：JDK 是 Java 开发工具包，提供了编写、调试和运行 Java 程序所需的工具。它包括了开发 Java 应用程序的所有工具，例如编译器 (`javac`)、调试工具 (`jdb`)、以及 JRE 和其他开发工具。
         - **包含内容**：
           - JRE（Java Runtime Environment）
           - Java 编译器（`javac`）
           - 其他开发工具（如 `javadoc`，`jarsigner` 等）
      2. **JRE（Java Runtime Environment）**：
         - **作用**：JRE 是 Java 运行时环境，用于运行已编译的 Java 程序。它包括了 Java 的标准库以及 JVM，负责为 Java 程序提供运行时支持。
         - **包含内容**：
           - JVM（Java Virtual Machine）
           - 核心类库（如 `java.lang` 包中的类）
           - 其他运行时工具
      3. **JVM（Java Virtual Machine）**：
         - **作用**：JVM 是 Java 虚拟机，负责执行 Java 字节码。它是 Java 应用程序能够跨平台运行的核心技术，JVM 根据不同的操作系统和硬件架构提供不同的实现。JVM 将 Java 字节码转换为特定平台上的机器码来执行。
         - **关键功能**：
           - 加载字节码
           - 执行字节码指令
           - 垃圾回收
           1. - - 
      
      ### 关系总结：
      
      - **JDK** 包含 **JRE**，而 **JRE** 包含 **JVM**。JDK 是开发工具包，JRE 是运行环境，而 JVM 是执行 Java 字节码的虚拟机。```

4. 环境变量path配置及其作用`Homework04.java`

   1. ```
      环境变量path及其作用：\n配置略，作用：命令查找：当你在终端（命令行）中输入命令时，系统会根据 PATH 环境变量指定的路径顺序，依次在不同的文件夹中查找是否有与命令名相匹配的可执行文件。如果找到了，就执行该文件。 方便快捷：通过配置 PATH，你不需要每次都输入完整的文件路径。只需要输入命令名，系统就能找到并执行相应的程序。
      ```
