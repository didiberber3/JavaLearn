03

装箱 (Boxing) 与 拆箱 (Unboxing)

手动装箱（JDK5之前）

// 方式一：构造器（已过时，Java 9+ 标记为 @Deprecated）
Integer i1 = new Integer(10);

// 方式二：静态工厂方法（官方推荐）
Integer i2 = Integer.valueOf(10);

手动拆箱

// 调用 xxxValue() 系列方法
int n = i1.intValue();
double d = i2.doubleValue();
byte b = i1.byteValue();

自动装箱 & 自动拆箱（JDK5+）⭐重点

Integer i3 = 100;           // 自动装箱 → Integer.valueOf(100)
int n2 = i3;               // 自动拆箱 → i3.intValue()

// 运算时自动拆箱
Integer a = 10;
Integer b = 20;
int sum = a + b;           // 自动拆箱→int，相加后→结果int
装箱过程：
基本类型 int ——valueOf()——→ 包装对象 Integer
拆箱过程：
包装对象 Integer ——intValue()——→ 基本类型 int
⚠️ 高频NPE陷阱：当 Integer 为 null 时进行自动拆箱会抛出 NullPointerException！
04

⭐⭐⭐ 重难点：Integer 缓存机制

缓存范围：-128 ~ 127

Integer.valueOf() 内部维护了一个缓存数组，当值在 -128 到 127 之间时，直接返回缓存中的同一个对象。

Integer m = 1;      // Integer.valueOf(1)→ 取缓存对象
Integer n = 1;      // Integer.valueOf(1)→ 取同一个缓存对象
System.out.println(m == n);   // ✅ true (同一对象)

Integer x = 128;    // Integer.valueOf(128)→ 超出缓存，new新对象
Integer y = 128;    // Integer.valueOf(128)→ 再次new新对象
System.out.println(x == y);   // ❌ false (不同对象)
原理：Integer.valueOf() 源码中维护了 IntegerCache 内部类， 默认缓存 [-128, 127] 共 256 个 Integer 对象。可通过 JVM 参数 -XX:AutoBoxCacheMax=size 调整上限。

各类型缓存范围一览

Wrapper类缓存范围说明
Integer-128 ~ 127上限可调，默认256个对象
Byte, Short, Long-128 ~ 127固定，不可调整
Character0 ~ 127ASCII范围
BooleanTRUE / FALSE只缓存两个静态常量
Float, Double❌ 无缓存浮点数精度高，不适合缓存
⚠️ 面试高频陷阱：永远用 .equals() 比较两个包装类对象的值，不要用 ==！ == 比较的是对象地址，不是值。
05

⭐⭐ 重难点：== 比较全场景分析

场景表达式结果原因
new vs new new Integer(127) == new Integer(127) false 两个独立堆对象，地址不同
new vs new(大值) new Integer(128) == new Integer(128) false 同上，与值大小无关
自动装箱(范围内) Integer x=127; Integer y=127; x==y true 缓存命中，同一对象
自动装箱(范围外) Integer x=128; Integer y=128; x==y false 超出缓存，创建不同对象
Integer vs int Integer(127) == 127 true 自动拆箱→基本类型值比较
Integer vs int(大值) Integer(128) == 128 true 同上，自动拆箱后值相等
Integer vs null Integer x=null; x==127 NullPointerException 自动拆箱null.intValue()
核心记忆法则：
① Wrapper==Wrapper：比较地址（受缓存影响）
② Wrapper==基本类型：触发自动拆箱，比较值
③ 基本类型==基本类型：直接比较值
④ 面试/生产环境：一律用.equals()
06

⭐⭐⭐ 重难点：三元运算符类型统一陷阱

// 陷阱示例
Object obj = true ? new Integer(1) : new Double(2.0);
System.out.println(obj);   // 输出：1.0 (不是 1！)
System.out.println(obj.getClass());  // class java.lang.Double
原因：三元运算符 ? : 会将两个分支的表达式统一为一个公共类型。 当一边是 Integer，另一边是 Double 时，Java 会进行二进制数值提升 (Binary Numeric Promotion)， Integer 被自动提升到精度更高的 Double。所以 new Integer(1) 变成了 1.0。

对比：if-else 不受影响

Object obj2;
if (true) {
    obj2 = new Integer(1);   // 各自独立评估
} else {
    obj2 = new Double(2.0);
}
System.out.println(obj2);              // 输出：1 ✅
System.out.println(obj2.getClass());   // class java.lang.Integer
✅ 避坑指南：三元运算符的使用场景中，确保两个分支类型一致（如都要是 int 或都要是 double），否则会发生隐式类型提升，导致预期之外的精度变化。
07

Wrapper ↔ String 相互转换

Wrapper / 基本类型 → String（3种方式）

Integer i = 100;

// 方式1：字符串拼接（最简便，但底层用StringBuilder）
String s1 = i + "";
String s2 = i + "大力";

// 方式2：调用 toString() 方法
String s3 = i.toString();         // Integer.toString()
String s4 = Integer.toString(100);
String s5 = Integer.toBinaryString(100);  // "1100100" 进制转换

// 方式3：String.valueOf()
String s6 = String.valueOf(i);
String s7 = String.valueOf(100);

String → Wrapper / 基本类型

String str = "123";

// 方式1：parseXxx() 静态方法（返回基本类型）
int n1 = Integer.parseInt(str);
double n2 = Double.parseDouble("3.14");
boolean n3 = Boolean.parseBoolean("true");

// 方式2：valueOf() 静态方法（返回包装对象）
Integer n4 = Integer.valueOf(str);

// 方式3：构造器（已过时，不推荐）
Integer n5 = new Integer(str);
parseXxx vs valueOf：
parseInt() 返回 int（基本类型），valueOf() 返回 Integer（包装对象，享受缓存）。

终极总结：八大Wrapper重难点速记

#知识点一句话总结难度
1八大类名 Byte Short Integer Long Float Double Boolean Character 基础
2父类区别 6个数字型继承 Number；Boolean 和 Character 直接继承 Object 重要
3装箱机制 Integer i = 100; 编译器自动调用 Integer.valueOf(100) 重要
4拆箱机制 int n = i; 编译器自动调用 i.intValue() 重要
5Integer缓存 -128~127 范围内自动装箱返回同一对象；范围外创建新对象 ⭐难点
6== vs equals == 比较地址（受缓存干扰）；.equals() 比较值——面试/生产请用后者 ⭐难点
7三元类型统一 true ? Integer(1) : Double(2.0) 结果是 Double 1.0，精度被提升 ⭐难点
8NPE陷阱 Integer i = null; int n = i; 自动拆箱抛 NullPointerException ⭐难点
9String互转 parseInt()→基本类型；valueOf()→包装对象；+""/toString()→String 重要
10缓存差异 Float/Double 无缓存；Boolean 只缓存 TRUE/FALSE 两个常量 重要
📌 面试一句话：Wrapper 的本质是"让基本类型也能当对象用"。 核心三件事——装箱拆箱（valueOf / xxxValue）、缓存机制（-128~127）、 类型互转（parseXxx / valueOf / toString）。 记住这三点 + 善用 .equals()，Wrapper 没有坑。
